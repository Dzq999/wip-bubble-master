# Flow 07 Business Rules

## 目标

Flow 07 将 Flow 06 的候选原因转化为可协同推进的工程问题包与任务清单。它服务于跨部门并行处理，不负责验证处置是否有效。

## 任务拆分原则

- 任务必须绑定同一个 Case，避免每个角色独立开散点问题。
- 每个任务需要明确角色、动作、输入证据、预期反馈、SLA 和完成门禁。
- 任务必须围绕 Flow 06 的候选原因与 Flow 04/05 的影响范围，不添加前端 demo 或 SQL 中不存在的业务指标。
- 对临时措施的任务只能写“继续执行 / 复核 / 反馈”，不能写“已经完成恢复”。
- 恢复指标需要给 Flow 08 验证，例如 WIP ratio、Queue、Move-Out、Tool Active、下游 Starvation、Hot Lot / Super Hot Run 风险。

## 禁止输出

- 不得声称已真实派单、已通知具体人员或外部系统已执行。
- 不得宣布处置已生效或异常已恢复。
- 不得宣布最终根因已确认。
- 不得关闭 Case。
