# Flow 08 Business Rules

## 目标

Flow 08 用于确认 Flow 07 的协同处置是否产生初步效果，并决定是否进入 Flow 09 影响消除观察。它不是 Case 关闭流程。

## 恢复口径

- 默认假设处置反馈已返回并完成初步确认。
- 异常数据恢复无需 SQL 或额外脚本，使用动态预置值。
- 只对前序中已异常的指标生成恢复值；原本不异常的指标保持原值并标记 Stable。
- Actual WIP 若高于 Target WIP，则恢复值取略低于 Target WIP。
- `Case Risk Trend｜处置后（恢复趋势）` 必须展示 `异常值 -> 正常值`，并通过 `meta.tone` 标记 `down`、`up` 或 `complete`。

## 判断逻辑

- 若角色反馈齐备，且 WIP / Queue / Move-Out / Tool / Downstream / Hot Lot 风险均无继续恶化信号，可判定为“初步有效”，进入 Flow 09。
- 若输入明确显示部分指标改善但仍缺关键反馈，判定为“部分有效”，要求继续补证或保留 On Hold。
- 若输入明确缺少处置反馈或恢复指标，判定为“证据不足”，不得进入 Flow 09。
- 若指标继续恶化或临时措施失控，判定为“需回退/升级”，要求回到协同处置层面调整任务包。

## 禁止输出

- 不得写“异常已完全恢复”。
- 不得写“Case 可关闭”。
- 不得写“已完成复盘”。
- 不得把 Flow 07 的任务建议描述成已经真实执行完成。
